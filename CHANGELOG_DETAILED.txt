0.2 ANPASSUNGEN AN MAINWINDOW
================================================================================

PROBLEM:
- MainWindow aus uebung10 erstellt ein sichtbares SDL-Fenster
- Für Qt/QML-Integration benötigen wir ein verstecktes Fenster
- Game-Loop muss extern steuerbar sein

ÄNDERUNGEN:

Datei: project1/src/game/MainWindow.cpp
Zeile: 134-145

VORHER (uebung10):
    m_window = SDL_CreateWindow(
        "Jumper Main Window",
        SDL_WINDOWPOS_UNDEFINED,
        SDL_WINDOWPOS_UNDEFINED,
        m_width,
        m_height,
        SDL_WINDOW_SHOWN  // Sichtbares Fenster
    );
    
    m_renderer = SDL_CreateRenderer(m_window, -1, 0);

NACHHER (project1):
    m_window = SDL_CreateWindow(
        "Jumper Main Window",
        SDL_WINDOWPOS_UNDEFINED,
        SDL_WINDOWPOS_UNDEFINED,
        m_width,
        m_height,
        SDL_WINDOW_HIDDEN  // Verstecktes Fenster!
    );
    
    m_renderer = SDL_CreateRenderer(m_window, -1, SDL_RENDERER_SOFTWARE);

BESCHREIBUNG:
- SDL_WINDOW_SHOWN → SDL_WINDOW_HIDDEN: Fenster wird nicht angezeigt
- SDL_RENDERER_SOFTWARE: Software-Renderer für bessere SDL_RenderReadPixels-Kompatibilität

Datei: project1/src/game/MainWindow.hpp
Zeile: Neue Methoden hinzugefügt

VORHER (uebung10):
    void run();  // Haupt-Game-Loop

NACHHER (project1):
    void run();  // Haupt-Game-Loop (wird nicht mehr verwendet)
    void update(const Uint8* keystates);  // Externes Update
    void render();  // Externes Rendering
    SDL_Renderer* renderer();  // Zugriff auf Renderer
    Level* level();  // Zugriff auf Level

BESCHREIBUNG:
- run() existiert noch, wird aber nicht mehr verwendet
- update() und render() ermöglichen externe Steuerung
- renderer() gibt Renderer für Pixel-Kopie zurück
- level() gibt Level-Zugriff für GameView

Datei: project1/src/game/MainWindow.cpp
Zeile: 92-107

HINZUGEFÜGT:

================================================================================
0.7 GAMEVIEW: BRÜCKE ZWISCHEN QT/QML UND SDL2
================================================================================

PROBLEM:
- SDL2-Spiel muss in Qt/QML eingebettet werden
- Rendering muss von SDL2 zu Qt/QML konvertiert werden
- Tastatur-Events müssen von Qt zu SDL konvertiert werden

LÖSUNG: GameView-Klasse

Datei: project1/GameView.hpp / GameView.cpp

FUNKTIONALITÄT:

1. SDL2-Initialisierung:
   - Initialisiert SDL2 mit verstecktem Fenster
   - Erstellt MainWindow-Instanz (aus uebung10)
   - Software-Renderer für Pixel-Kopie

2. Rendering-Pipeline:
   - MainWindow::render() → SDL-Renderer
   - SDL_RenderReadPixels() → SDL_Surface
   - SDL_Surface → QImage
   - QImage → QPainter (in paint())

3. Tastatur-Event-Konvertierung:
   - Qt KeyEvents → SDL Events
   - Qt KeyCodes → SDL Keycodes/Scancodes
   - Manuelle Key-State-Verwaltung (QMap)

4. Update-Loop:
   - QTimer mit 60 FPS (16ms Intervall)
   - Ruft MainWindow::update() auf
   - Triggert paint() für Rendering

WICHTIGE METHODEN:

- startGame(): Erstellt MainWindow, startet Timer
- stopGame(): Stoppt Timer, löscht MainWindow
- updateGame(): Update-Loop (SDL_PumpEvents, MainWindow::update)
- paint(): Rendering (SDL → QImage → QPainter)
- keyPressEvent/keyReleaseEvent: Qt → SDL Event-Konvertierung

================================================================================
0.9 RENDERING-PIPELINE IM DETAIL
================================================================================

SCHRITT 1: SDL2-Rendering
-------------------------
MainWindow::render() wird aufgerufen:
  - SDL_RenderClear()
  - Level::render() → LayerManager::render()
  - LayerManager setzt Camera-Offset für alle Renderables
  - Alle Renderables rendern relativ zur Kamera
  - SDL_RenderPresent()

SCHRITT 2: Pixel-Kopie
----------------------
SDL_RenderReadPixels() kopiert Pixel vom Renderer:
  - SDL_Surface wird erstellt (ARGB8888 Format)
  - Pixel werden vom Renderer in Surface kopiert
  - Surface enthält das komplette gerenderte Bild

SCHRITT 3: QImage-Konvertierung
--------------------------------
SDL_Surface → QImage:
  - QImage wird aus Surface-Pixeln erstellt
  - Format: QImage::Format_ARGB32
  - Tiefe Kopie (copy()), da Surface-Speicher freigegeben wird

SCHRITT 4: QML-Darstellung
--------------------------
QPainter::drawImage():
  - QImage wird in boundingRect() gezeichnet
  - QPainter rendert in QML-Kontext
  - Antialiasing deaktiviert für pixel-perfekte Darstellung

0.12 WICHTIGE ARCHITEKTUR-ENTSCHEIDUNGEN
================================================================================

1. VERSTECKTES SDL-FENSTER:
   - SDL2 benötigt ein Fenster für den Renderer
   - Lösung: Verstecktes Fenster (SDL_WINDOW_HIDDEN)
   - Renderer wird für Pixel-Kopie verwendet

2. SOFTWARE-RENDERER:
   - SDL_RenderReadPixels() funktioniert besser mit Software-Renderer
   - Hardware-Renderer könnte Probleme bei Pixel-Kopie haben
   - Lösung: SDL_RENDERER_SOFTWARE Flag

3. EXTERNE GAME-LOOP:
   - uebung10: MainWindow::run() (interne Loop)
   - project1: GameView::updateGame() (externe Loop über QTimer)
   - Ermöglicht Qt/QML-Integration

4. PIXEL-KOPIE STATT TEXTUR-SHARING:
   - SDL2-Texturen können nicht direkt in Qt verwendet werden
   - Lösung: Pixel-Kopie bei jedem Frame
   - Trade-off: Performance vs. Einfachheit

5. MANUELLE KEY-STATE-VERWALTUNG:
   - SDL_GetKeyboardState() benötigt SDL_PumpEvents()
   - Qt-Events müssen manuell zu SDL-Events konvertiert werden
   - Lösung: QMap für Key-States + SDL_Event-Pushing

    // Verhindere, dass der Spieler außerhalb des Levels läuft

    {
        // Berechne die Level-Grenzen in Welt-Koordinaten
        // Level startet bei (0, 600) in Welt-Koordinaten (Y-Offset für weiter unten)
        int levelLeft = 0;
        int levelRight = m_tiles->width() * m_tiles->tileWidth();
        
        // Prüfe linke Grenze
        if(desiredPosition.x() < levelLeft)
        {
            desiredPosition.setX(levelLeft);
            // Stoppe horizontale Geschwindigkeit, wenn gegen linke Wand
            if(m_actor->velocity().x() < 0)
            {
                m_actor->velocity().setX(0);
            }
        }
        
        // Prüfe rechte Grenze (berücksichtige Sprite-Breite)
        if(desiredPosition.x() + m_actor->w() > levelRight)
        {
            desiredPosition.setX(levelRight - m_actor->w());
            // Stoppe horizontale Geschwindigkeit, wenn gegen rechte Wand
            if(m_actor->velocity().x() > 0)
            {
                m_actor->velocity().setX(0);
            }
        }
    }
